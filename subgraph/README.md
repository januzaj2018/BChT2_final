# GameFi Protocol Subgraph

This subgraph indexes all on-chain events generated by the GameFi Protocol smart contracts deployed on **Arbitrum Sepolia**. It provides efficient, low-latency, and custom-structured data queries for the React dApp using GraphQL.

## Supported Entities

The subgraph defines 7 entities in `schema.graphql` covering the full range of protocol activities:
1. **GameItem**: Tracks user balances of ERC-1155 game resources/items (materials IDs 1-10).
2. **Craft**: Logs successful crafting operations, including recipe IDs and inputs/outputs.
3. **AMMLiquidity**: Tracks user liquidity provisioning and pool shares in the constant-product AMM.
4. **Swap**: Records resource swaps with swap directions, amounts, and exact timestamps.
5. **VaultDeposit**: Tracks NFT stakings/deposits into the ERC-4626 Rental Vault.
6. **Proposal**: Indexes OpenZeppelin governance proposals, descriptions, and current status.
7. **Vote**: Logs voting actions with support types, voting weight, and optional text reasons.

## 1. Setup and Build Instructions

### Prerequisites
Ensure you have the Graph CLI installed globally:
```bash
npm install -g @graphprotocol/graph-cli
```

### Install Dependencies
Run the following from the `gamefi-protocol/subgraph/` directory:
```bash
npm install
```

### Code Generation
Compile the GraphQL schema and contract ABIs into TypeScript types:
```bash
npm run codegen
```

### Build Subgraph
Compile the AssemblyScript mappings into WASM:
```bash
npm run build
```

### Deploy to The Graph Studio
1. Authenticate with your Graph Studio Deploy Key:
   ```bash
   graph auth --studio <YOUR_DEPLOY_KEY>
   ```
 2. Deploy to the hosted service:
   ```bash
   npm run deploy
   ```

## 2. Documented GraphQL Queries

Here are **5 essential GraphQL queries** used to fetch data for the protocol's frontend:

### Query 1: Fetch User Inventory (GameItem balances)
Retrieves the balances of all ERC-1155 resources owned by a specific user.
```graphql
query GetUserInventory {
  gameItems(where: { owner: "0xb9b59714052a56ab926cb7b30e0e31a3ab67ec5f" }) {
    id
    tokenId
    balance
  }
}
```

### Query 2: Fetch Recent Swaps (AMM Trading Feed)
Fetches the last 10 resource swap events in the GameAMM, showing active traders and transaction values.
```graphql
query GetRecentSwaps {
  swaps(first: 10, orderBy: timestamp, orderDirection: desc) {
    id
    user
    tokenIn
    amountIn
    tokenOut
    amountOut
    timestamp
  }
}
```

### Query 3: Get All Governance Proposals
Queries the full list of proposals submitted to the DAO, including their descriptions and status.
```graphql
query GetGovernanceProposals {
  proposals(orderBy: startBlock, orderDirection: desc) {
    id
    proposer
    description
    status
    startBlock
    endBlock
  }
}
```

### Query 4: Get Vote Cast History for a Proposal
Fetches the voting history for a specific proposal ID, detailing voting choices and voting weights.
```graphql
query GetProposalVotes {
  votes(where: { proposalId: "402830843209590823528" }) {
    id
    voter
    support
    weight
    reason
  }
}
```

### Query 5: Fetch Rental Vault Deposit History
Retrieves staker deposit history to display lifetime statistics in the Rental Vault interface.
```graphql
query GetVaultDeposits {
  vaultDeposits(orderBy: timestamp, orderDirection: desc) {
    id
    user
    tokenId
    amount
    shares
    timestamp
  }
}
```
